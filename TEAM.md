  - Использованы `pytest` + `pytest-asyncio` для асинхронных методов. 

- **Тройная защита данных**  
  - Два независимых зеркала (`storage/vault_a.bin`, `storage/vault_b.bin`) с разными XOR‑ключами.  
  - Основная БД SQLite (`storage/xbank.db`).  
  - При запуске сравниваются все три источника + проверяется `keyring`.  
  - Любое расхождение вызывает критическую ошибку — подмена баланса невозможна.

## Алексей  
**Коммит:** `Aleksei/iter2: PySimpleGUI wallet & Bluetooth P2P`

### Добавленные / изменённые файлы:
- `main.py` – графическое приложение на `PySimpleGUI`  
- `scanner.py` – сканирование Bluetooth‑устройств через `bleak`  
- `server.py` – GATT‑сервер на `bless` для приёма переводов  
- `transfer.py` – отправка транзакций через BLE  
- `protocol.py` – упаковка/распаковка JSON‑транзакций  
- `manager.py`, `models.py`, `database.py` – доработаны под GUI и асинхронность  
- `docker-compose.yml` – контейнеризация с пробросом X11 для GUI

### Что реализовано:
- **Графический интерфейс (BlippyApp)**  
  - Главное окно с отображением адреса, баланса, списка найденных узлов.  
  - Кнопка «Найти получателей» запускает Bluetooth‑сканирование.  
  - Перевод: выбор пользователя из списка, ввод суммы, отправка через `transfer.py`.  
  - Кнопка «История» показывает все транзакции в popup‑окне.  
  - Полная асинхронная интеграция: `asyncio` + `PySimpleGUI` (события через `write_event_value`).  

- **Bluetooth‑кошелёк**  
  - **Сканер** (`scanner.py`): обнаруживает устройства с префиксом `node-` и RSSI > -80.  
  - **Сервер** (`server.py`): GATT‑сервер с кастомным сервисом/характеристикой. При получении данных вызывает callback на обновление баланса.  
  - **Передача** (`transfer.py`): подключается к устройству по BLE, отправляет упакованную транзакцию.  

- **Docker‑контейнеризация**  
  - `docker-compose.yml` поднимает контейнер с пробросом X11, `dbus` и `.Xauthority`.  
  - Позволяет запускать GUI‑приложение в изолированной среде.
